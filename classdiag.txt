Compositions (◆ filled diamond on the OWNER end):

  [Composition]  User (1) ◆─── owns ─── (0..*) Workspace
    A user owns one or more workspaces. If the owner is deleted,
    all owned workspaces are deleted.

  [Composition]  Workspace (1) ◆─── contains ─── (0..*) Invoice
    A workspace contains all its invoices. Invoices cannot exist
    outside a workspace.

  [Composition]  Workspace (1) ◆─── hosts ─── (0..*) ChatConversation
    All channels and DM threads belong to a workspace.

  [Composition]  Workspace (1) ◆─── logs ─── (0..*) ActivityLog
    Activity log entries belong to the workspace they were recorded in.

  [Composition]  Workspace (1) ◆─── issues ─── (0..*) Invitation
    Join/leave/renewal requests are scoped to a workspace.

  [Composition]  Invoice (1) ◆─── stores ─── (0..*) Document
    Documents are attachments of an invoice; they are deleted with it.

  [Composition]  Invoice (1) ◆─── extracts ─── (0..*) ExtractedField
    OCR-extracted fields belong to exactly one invoice.

  [Composition]  Invoice (1) ◆─── tracks ─── (0..*) StatusHistory
    Status change records are owned by their invoice.

  [Composition]  Company (1) ◆─── holds ─── (0..*) Subscription
    A company can have multiple subscriptions (current + history).

  [Composition]  User (1) ◆─── receives ─── (0..*) Notification
    Notifications are created for a specific user and deleted with them.

  [Composition]  ChatConversation (1) ◆─── contains ─── (0..*) ChatMessage
    Messages are part of a conversation; deleted when conversation
    is deleted.

Aggregations (◇ hollow diamond on the WHOLE end):

  [Aggregation]  Workspace (1) ◇─── has ─── (0..*) Membership
    A workspace aggregates its memberships. The Member (User) exists
    independently of any one workspace.

  [Aggregation]  ChatConversation (1) ◇─── includes ─── (0..*) ChatMember
    A conversation groups its member records. The User exists
    independently of any one conversation.

  [Aggregation]  Workspace (1) ◇─── belongs to ─── (1) Company
    A company workspace is described by exactly one Company record.
    Company cannot exist without its workspace (1:1 mapping).
    Diamond on Workspace end.

Associations (plain solid lines ────):

  [Association]  User (1) ─── joins via ─── (0..*) Membership
    A user has zero or more membership records across workspaces.

  [Association]  Membership (0..*) ─── assigned ─── (1) Role
    Each membership specifies exactly one role for that user in
    that workspace.

  [Association]  Role (1) ─── defines ─── (0..*) Invitation
    An invitation specifies the role to be assigned to the
    requesting user.

  [Association]  User (1) ─── creates ─── (0..*) Invoice
    A user (Employee/Director/Personal) creates invoices.
    (created_by — set to NULL if user is deleted, invoice preserved)

  [Association]  User (1) ─── uploads ─── (0..*) Document
    A user uploads document attachments.
    (uploaded_by — set to NULL if user is deleted)

  [Association]  User (1) ─── corrects ─── (0..*) ExtractedField
    A user manually overrides an OCR-extracted field value.
    (corrected_by — set to NULL if user is deleted)

  [Association]  User (1) ─── records ─── (0..*) StatusHistory
    A user triggers each status change entry.
    (changed_by — set to NULL if user is deleted)

  [Association]  User (1) ─── starts ─── (0..*) ChatConversation
    A user creates channels or DM threads.
    (created_by — set to NULL if user is deleted)

  [Association]  User (1) ─── participates in ─── (0..*) ChatMember
    A user is a member of zero or more chat conversations.

  [Association]  User (1) ─── sends ─── (0..*) ChatMessage
    A user sends chat messages.
    (sender_id — message deleted if user is deleted)

  [Association]  User (1) ─── tracks activity ─── (0..*) ActivityLog
    Activity log entries reference the user who performed the action.
    (user_id — set to NULL if user is deleted)

  [Association]  SubscriptionPlan (1) ─── defines terms of ─── (0..*) Subscription
    Each subscription follows one subscription plan.

  [Association]  User (1) ─── subscribes personally ─── (0..1) Subscription
    A user can hold one personal workspace subscription.

Dependency (dashed arrow - - ->):

  [Dependency]  Invoice - - - uses - - -> InvoiceStatus
    Invoice.currentStatus references a valid InvoiceStatus value.
    Shown as a dependency because InvoiceStatus is an enumeration
